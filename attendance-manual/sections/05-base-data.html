基礎資料管理

管理員 — 基礎資料管理涵蓋公司核心設定，建議由管理員（人事）進行日常維護。

公司設定

設定公司基本資訊及打卡相關參數。

設定項目說明

欄位 說明
公司名稱 公司正式名稱（必填）
地址 公司地址
GPS 緯度 / 經度 公司所在的 GPS 座標，作為打卡圍欄的中心點
圍欄半徑（公尺） GPS 打卡圍欄的有效半徑，預設 200 公尺
Wi-Fi SSID 公司指定的 Wi-Fi 名稱，員工打卡時必須連接此 Wi-Fi 才算正常打卡
LINE Channel ID LINE Login 的 Channel ID（由 LINE Developers Console 取得）
LINE Channel Secret LINE Login 的 Channel Secret（留空表示不更改）
人事部門 指定人事部門，系統會用於特定流程的判斷

操作步驟

  1. 在左側選單點擊「基礎資料 > 公司設定」
  2. 修改需要變更的欄位
  3. 點擊「儲存設定」按鈕
公司設定 — 設定公司基本資訊、GPS 座標、Wi-Fi SSID 等
⚠️ 重要提醒 修改 Wi-Fi SSID 或 GPS 座標會直接影響員工的打卡驗證。請確認設定正確後再儲存。若公司未設定 Wi-Fi SSID，系統會退回純 GPS 模式驗證。

部門管理

管理公司的組織架構，建立、編輯部門並指定部門主管。

檢視部門列表

  • 頁面顯示所有部門：部門名稱、部門主管、狀態

新增部門

  1. 點擊「＋ 新增部門」按鈕
  2. 輸入部門名稱
  3. 選擇部門主管（從員工清單中選擇）
  4. 點擊「確定」儲存
部門管理 — 部門列表
新增部門 — 輸入名稱並指定主管

編輯 / 刪除部門

  • 點擊「編輯」可修改部門名稱和主管
  • 點擊「刪除」可移除部門；若部門下仍有員工，無法刪除，系統會提示「有 X 位員工在此部門，無法刪除」

員工管理

管理全公司員工資料，包含新增、編輯、停用員工，以及 LINE 綁定和代理人設定。

檢視員工列表

  • 可使用搜尋框以姓名或工號搜尋
  • 可依部門和狀態（在職 / 停用）篩選
  • 表格欄位：員工編號、姓名、部門、職稱、到職日、角色、LINE 綁定狀態、在職狀態
  • 操作：編輯、代理人設定、LINE 綁定碼、停用/啟用
員工管理 — 員工列表（可搜尋、篩選部門與狀態）

新增員工

  1. 點擊右上角「＋ 新增員工」按鈕
  2. 填寫必要資訊：員工編號、姓名、Email、部門、職稱、到職日
  3. 選擇角色：管理員 / 員工 / 老闆
  4. 選擇預設班別
  5. （選填）輸入月薪，系統將用於薪資輔助報表試算
  6. 設定初始密碼
  7. 點擊「確定」儲存
新增員工 — 填寫員工基本資料
⚠️ 注意事項
  • 員工編號不可重複，設定後無法修改
  • 到職日是計算年資與特休天數的依據，請確認正確
  • 角色設定為「老闆」的帳號將成為最終簽核人（全公司僅應有一位）

編輯員工

點擊員工列表中的「編輯」按鈕，可修改員工的部門、職稱、班別、月薪、密碼等資訊。若該員工不需打卡（如特殊主管職位），可開啟「免打卡員工」開關，系統將跳過差勤異常偵測。

代理人設定

為員工指定代理人，當該員工請假時，其待簽核的項目會由代理人協助處理。

  1. 在員工列表中，點擊對應員工的「代理人」按鈕
  2. 從下拉選單中選擇代理人
  3. 確認儲存

LINE 綁定

管理員可為員工產生 LINE 綁定碼，員工在手機 APP 首次使用 LINE 登入時，需輸入此綁定碼來關聯帳號。

  • 點擊「LINE 綁定」按鈕可產生綁定碼
  • 將綁定碼提供給員工，員工在 APP 中輸入即完成綁定
  • 已綁定的員工，表格中 LINE 欄位會顯示「已綁定」

停用 / 啟用員工

  • 點擊「停用」可將離職員工帳號標記為停用，停用後員工無法登入但歷史紀錄保留
  • 已停用的員工可點擊「啟用」恢復使用

班別設定

管理公司的班別（工作時間制度），每位員工會被指派一個預設班別。

班別類型

類型 說明 適用情境
固定班 設定明確的上下班時間，系統依此判斷遲到、早退 多數員工的標準工作制
彈性班 不設固定上下班時間，僅檢查每日最低工時 不需管控上下班時間的特殊職位

新增班別

  1. 點擊「＋ 新增班別」按鈕
  2. 輸入班別名稱（如「標準全天班」）
  3. 選擇班別類型（固定班/彈性班）
  4. 固定班需設定：上班時間、下班時間、彈性分鐘數
  5. 彈性班需設定：最低工時（小時）
  6. 可設定是否為預設班別
  7. 點擊「確定」儲存
班別設定 — 班別列表
新增班別 — 依類型設定不同參數
💡 彈性分鐘數說明 以標準全天班為例：上班時間 08:30、彈性時段至 09:35（09:35:59 前打卡均視為正常；09:30:01–09:35:59 視同 09:30 打卡），09:36 之後才算遲到。

假別設定

管理公司提供的假別種類及其規則。

假別列表

  • 顯示所有假別：名稱、年度上限天數、是否需附件、給薪方式、最小請假單位、簽核層級、啟用狀態
  • 可以切換啟用/停用假別（停用後員工無法申請該假別）
假別設定 — 假別列表及啟用狀態

新增假別

  1. 點擊「＋ 新增假別」按鈕
  2. 填寫假別名稱
  3. 設定年度上限天數
  4. 設定最小請假單位（0.5 天 / 1 天 / 0.5 小時 / 1 小時等）
  5. 設定是否需要附件證明
  6. 設定簽核層級（1 = 僅需部門主管 / 2 = 主管 + 老闆）
  7. 點擊「確定」儲存
新增假別 — 設定假別規則
💡 給假方式說明
  • 員工可直接申請：如事假、病假、生理假、公差，系統自動提供額度，員工可直接在自助服務中申請
  • 後端給假：如婚假、喪假、產假等特殊假別，須由管理員先在「假期額度」中為員工建立額度，員工才能看到並申請

年資特休

設定年資與特休天數的對照規則，系統會依據員工到職日的年資自動計算特休天數。

年資規則列表

  • 顯示各年資區間對應的特休天數
  • 預設已依勞基法 §38 建立標準規則
  • 公司可設定優於法令的規則（如較多的特休天數）
年資特休 — 年資區間與特休天數對照表

新增 / 編輯規則

  1. 點擊「＋ 新增年資規則」
  2. 設定年資起始月數與年資結束月數
  3. 設定對應的特休天數
  4. 點擊「確定」儲存
新增年資規則 — 設定年資區間與天數
💡 年資計算說明 系統採曆年制（每年 1/1～12/31），到職首年依實際月份按比例折算。每年初系統會自動依據年資規則計算各員工的特休天數。

假期額度

管理各員工在各假別的年度額度。部分假別（如特休）可批次初始化，部分特殊假別（如婚假、喪假）需手動建立。

頁面說明

  • 選擇年度查看該年度所有員工的假期額度
  • 表格顯示：員工、假別、總天數、已使用、剩餘
  • 可直接在表格中編輯額度（如手動調整天數）
假期額度 — 選擇年度查看各員工的假別額度

初始化特休額度

  1. 選擇年度（如 2026 年）
  2. 點擊「初始化特休額度」按鈕
  3. 系統會依據「年資特休」規則與員工到職日，自動計算並建立所有在職員工的特休額度
初始化特休額度 — 一鍵依年資規則批次產生特休額度
⚠️ 注意 初始化特休額度會覆蓋已存在的特休額度。建議在每年初執行一次，若該年度已有特休資料請謹慎操作。

行事曆

管理公司的年度行事曆，包含國定假日、公司自訂假日及補班日的設定。

行事曆類型

類型 說明 對出勤的影響
國定假日 依政府公告之國定假日 當日不需上班，不列入出勤計算
公司假日 公司自訂假日（如颱風假、尾牙） 當日不需上班，不列入出勤計算
補班日 依政府公告之補班日 原本休假日改為上班日，需正常出勤

操作說明

  • 頁面以年份 Tab 切換不同年度的行事曆
  • 可新增假日：選擇日期、類型、輸入名稱
  • 可編輯或刪除現有假日
  • 支援批次匯入功能，可一次匯入整年度的國定假日
行事曆 — 以年份 Tab 切換，顯示國定假日、公司假日、補班日
新增假日 — 選擇日期、類型與名稱
💡 提醒 行事曆設定會直接影響出勤報表中的「應出勤天數」計算，以及假日加班的認定。建議每年初將當年度國定假日匯入系統。
