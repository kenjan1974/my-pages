二、為何目前可以用免費方案

2-1. Render（API 伺服器）

Render 的免費方案提供 Web Service 部署功能，適合以下情境：

  • 系統使用者為公司內部員工，非 24 小時對外開放服務
  • 員工打卡集中在上班時段（早上 9:00 前後），不需全天待命
  • 免費方案限制：服務在 15 分鐘無流量後自動休眠，第一個請求需 20–30 秒喚醒（冷啟動）
目前夠用的原因：差勤系統的打卡行為本身會觸發 API 請求，使伺服器保持喚醒。只要員工在上班時段打卡，冷啟動問題極少出現。

2-2. Supabase（資料庫）

Supabase 免費方案包含：

項目 免費額度
資料庫儲存空間500 MB
每月活躍使用者（MAU）50,000 人
檔案儲存空間1 GB
每月資料傳輸量（Egress）5 GB
專案暫停機制連續 7 天無活動自動暫停
目前夠用的原因：
  • 系統針對 20 人以下中小企業設計
  • 20 人全年差勤資料（打卡、請假、加班、通知）預估不超過 50 MB
  • 員工每天打卡會持續觸發資料庫，不會觸發「7 天無活動暫停」

2-3. Firebase FCM（推播通知）

Firebase Cloud Messaging 是 Google 永久免費的服務，無使用量上限，完全不會產生費用。

2-4. LINE Login

LINE Login API 由 LINE 官方提供，供第三方應用整合登入用途，永久免費，無使用量限制。

2-5. LINE Messaging API（開發 / 測試期間）

LINE Messaging API 需搭配 LINE 官方帳號（LINE Official Account）使用，系統用它將核准結果、異常通知等訊息主動推送至員工的 LINE。

LINE OA 免費的「輕用量方案」每月提供 200 則免費訊息，在開發測試期間（真實員工尚未使用）完全夠用。

三、何時需要升級付費方案

情境 A：上線初期（強烈建議）— 升級 Render

問題：Render 免費服務在無流量時會休眠，第一個打卡請求可能需等待 20–30 秒才有回應。雖然可用定時 Ping 方式維持喚醒，但此方法不穩定。

建議升級時機：系統正式上線、員工開始真實打卡時。

情境 B：上線第一天（幾乎確定需要）— 升級 LINE OA 方案

問題：系統在以下每個動作都會對相關人員發送 LINE 通知：
觸發動作 發送對象
員工提交請假 / 加班 / 補卡申請審核主管
主管核准 / 退回申請申請員工
員工撤銷申請審核主管
員工使用外勤打卡申請員工（確認通知）

20 人公司每月 LINE 訊息估算：

類型 每月申請數 訊息數（送出＋回覆）
請假申請約 40 件約 80 則
加班申請約 60 件約 120 則
補卡申請約 20 件約 40 則
外勤打卡確認約 100 次約 100 則
撤銷 / 其他—約 20 則
合計約 360 則 / 月
免費方案上限 200 則 / 月，正式上線後幾乎立即超額。輕用量方案超過上限後無法繼續發送，員工將收不到任何核准通知。

建議升級時機：正式上線、員工開始提交申請之前。

情境 C：資料量成長 — 升級 Supabase

以下任一情況發生時需升級：

觸發條件 說明
員工人數超過 50 人差勤資料累積速度加快，儲存空間消耗提升
系統使用超過 2–3 年歷史紀錄累積，資料庫接近 500 MB 上限
需要資料備份免費方案不提供任何備份機制，資料遺失風險存在
需要確保 7×24 穩定性免費方案專案可能被暫停，需付費方案保障不暫停

情境 D：需要正式 SLA 保障

如客戶業務需要平台提供正式服務等級保證（SLA），免費方案均不提供此服務，需升級至付費方案。

五、總結：費用時程建議

開發 / 測試期間 └── 全部免費方案 ── 月費 $0 正式上線（第一天） ├── 升級 LINE OA 中用量方案 ── NT$800 / 月 │ 原因：免費方案 200 則上限，正式上線後立即超額 └── 升級 Render Starter ── 約 NT$230 / 月（$7 USD） 原因：消除冷啟動，確保員工打卡順暢 上線初期合計：約 NT$1,030 / 月 上線 3–6 個月後（視資料量與穩定性需求） └── 追加 Supabase Pro ── 約 NT$800 / 月（$25 USD） 原因：啟用每日備份、保障資料庫不被暫停 長期穩定運行 └── 合計約 NT$1,830 / 月

六、永久免費、無需額外費用的項目

以下服務無論使用量或上線狀態，均不產生費用：

  • Firebase FCM 推播通知 — Google 官方永久免費，無訊息數量限制
  • LINE Login — LINE 官方永久免費，無用量限制（注意：這是「登入功能」，與「訊息推送」的 LINE Messaging API 是不同服務，費用也分開計算）
